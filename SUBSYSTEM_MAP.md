## 四、定位差異化

| 入口 | 形態 | 目標使用者 | 使用情境 |
|------|------|-----------|----------|
| **小校一表通**（本專案）| Streamlit web app | 校內現場（總務主任、行政幹事）| 日常作業：問採購法、生草稿、整理會議 |
| `/校事文書` skill | Claude Code CLI | 教導主任（高敏感案件）| 校事會議文書產出 |
| `/採購法` skill | Claude Code CLI | 採購承辦人（深度研究）| 條文探究、決標分析 |
| `/關懷 立案/結案/發佈` skill | Claude Code CLI | 計畫承辦人 | S12/S13 跨子系統入口 |
| 6 個 UI 原型 HTML | 靜態 HTML（vault 根目錄）| 校長、總務主任 mock | UX 設計參考（總入口 Portal、校長儀表板 等）|

**為什麼 web app 跟 CLI skill 並存**：
- web app 給「**現場使用者**」（不會用 CLI 的同事）
- CLI skill 給「**深度使用者**」（校長 / 主任 / 候用校長自己）
